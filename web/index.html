  • Calculadora de IMC

  • Programa de entrada e saída

    Dificuldade: Fácil

  • Comparador de valores - Dois valores

    Compara dois valores com if e variáveis, traz a mensagem informando qual valor é maior ou menor
    Dificuldade: Fácil

  • Calculadora simples (2 entradas)

    Calculadora básica, com operações simples e apenas cálculo único
    Dificuldade: Fácil

  • Programa de entrada e saída com WHILE

    Programa de entrada e saída com variável de senha fixa
    Dificuldade: Médio

  • Exercício de índices - fatiando STRs

    Fatiamento com índices [::], contagem de caracteres com len(), if para condições básicas e entradas inválidas

    Clacula caracteres dentro de uma str, utilizando while e else
    Dificuldade: Médio

  • Mutação de str - iteração

    insere "*" entre os caracteres de uma str, modelando uma nova palavra, exemplo: *l*e*a*n*d*r*o.
    Dificuldade: Médio

    Programa para calcular média de notas, ele tras a média da nota total, a maior e menor nota.
    Dificuldade: Médio

  • Calcular caracteres dentro de uma STR

    Clacula caracteres dentro de uma str, utilizando while e a memória do próprio programa
    Dificuldade: Médio-alto / Difícil

  • Calcularadora com while, soma de operações e funcionalidades extras.

    Calculadora mais complexa que a anterior, calcula a quantidade de operações enquanto você faz adições aos calculos. Mostra também a opção de zerar o cálculo em andamento e sair da calculadora
    Dificuldade: Médio-alto / Difícil

  • Programa entrada e saída - Autenticação de usuário

    Programa de entrada e saída com autenticação de usuários e cadastro em memória, cadastro de senha e login. Contagem de tentativas e saída se excedido.
    Dificuldade: Médio-Alto / Difícil

  • Jogo da Forca

    Simula jogo da forca em Python, foram utilizados loops para criar str e comparar.
    Dificuldade: Médio → Médio/Avançado

Storytelling

  1. Início da jornada — 02/2026

    Iniciei meus estudos em programação com foco em migrar para a área de dados dentro da empresa. Já possuía uma noção básica de lógica de programação por estudos anteriores. Comecei com Python e paralelamente iniciei HTML e CSS para documentar meus estudos.

  2. Primeiros fundamentos

    Estudo de tipos de dados, estruturas condicionais, entrada e saída e laços de repetição. Desenvolvimento de exercícios básicos como calculadora de IMC e programas com validação de entrada.

  3. Primeiro contato com lógica de sistema (Dia 55)

    Desenvolvimento de sistema simples de autenticação com validação de usuário e senha utilizando while e if. Primeiro contato com lógica de funcionamento de sistemas reais.

  4. Evolução em loops e manipulação de dados (Dia 61+)

    Introdução ao for in, manipulação de índices e strings, contadores e acumuladores. Exercícios envolvendo lógica com repetição e tratamento de dados.

  5. Introdução ao front-end (Dia 62)

    Tentativa de recriação da interface inicial da Netflix utilizando HTML e CSS. Aprendizado em layout com flexbox, alinhamento e organização de conteúdo.

  6. Projeto: Jogo da Forca (Dia 63)

    Desenvolvimento de jogo completo com controle de estado, validação de entradas e uso de loops. Principal aprendizado: strings são imutáveis e devem ser reconstruídas a cada interação. Dificuldade: Médio → Médio/Avançado.

  7. Consolidação de lógica e listas (Dia 70 — 16/04/2026)

    Estudo de listas com append, pop e manipulação por índice. Desenvolvimento de sistemas como lista de compras e carrinho de compras com controle de fluxo, validações e interação com usuário.

  8. Aprimoramento Hub de estudos (Dia 71 - 17/04/2026)

    Substituído os divs do site estruturado para anotações e registros de estudos, por tags nav para abas de navegação. A lista de exercícios que era separada por divs agora foi estruturada em listas não ordenadas.

  9. Estudo de exercício PY e início de organização em git e github (Dia 79 - 24/04/2026)

    Realizado um exercício de média de notas com py, onde o usuário insere as notas e o programa mostra média, maior e menor nota; Início em git e github, reorganizando projetos, exercícios e estudos.

  10. Estruturação do projeto e padronização de CSS (Dia 82 — 27/04/2026)

    Estruturação do projeto "my-dev-journey" com organização de pastas voltada para versionamento no GitHub, separando assets, páginas e estilos para melhor escalabilidade e manutenção.
    Externalização do CSS, removendo estilos inline e centralizando a estilização em arquivos dedicados, promovendo reutilização de código e padronização visual entre as páginas.
    Primeiro contato prático com problemas reais de front-end, como caminhos relativos de arquivos (CSS e imagens) e impacto da estrutura de diretórios no funcionamento da aplicação.
    Principal aprendizado: importância da organização de projeto e separação de responsabilidades (HTML, CSS e estrutura de arquivos).
