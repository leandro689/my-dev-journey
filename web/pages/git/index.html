Estudos Git

Anotações

Repositório: É onde o codigo vai ser armazenado. (projeto)
.gitignore: Arquivo recomendado pra informarmos arquivos ou pastas sensíveis que por segurança é melhor deixar fora do repositório. Podemos destacar pastas e categorias de arquivos que devem ser ignoradas pelo git.

Comandos

git status Mostra o status atual do git, vai mostrar se tem arquivo pra commit, se tem arquivo pra salvar e etc
git init Inicia um novo projeto git
git add Adiciona o arquivo no git
git add . Adiciona todos os arquivos dentro do projeto de uma vez só
git commit Comita os arquivos do projeto Flag -m para adicionar msg- Importante comentar o que você fez com esse comit
git commit -a Comita varias arquivos de uma vez Flag -m para adicionar msg- Importante comentar o que você fez com esse comit
git remote mostra a origem do repo
git push salva o arquivo no repo do github git pull Puxa as atualizações do projeto que subiram no GitHub, geralmente utilizado em projetos em grupo, ou buscando de outra máquina
git clone Clona um repositório do GitHub, geralemte usamos quando ingressamos em um novo projeto.
git rm Deleta o arquivo do projeto, esse comando não isola o arquivo do monitoramento, ele realmente excluí do projeto.
git log Mostra as informações dos commits realizados no projeto.
git mv Permite mover e alterar o nome do arquivo. Funciona de forma similar ao ctr+x que tira o arquivo e deixa na memória até a fase de colar.
git checkout volta ao estado anterior. Precisa tomar atenção, pois depois disso não podemos utilziar o ctrl+z.
O git checkourt pode ser meio confuso, então o Git separou a função em git switch para trocar a branch, git swith -c para criar nova branch e git restore para restaurar o arquivo.
git reset --hard Volta o arquivo no tempo e apaga tudo o que foi feito. Pode-se utilizar o id do commit da versão estável para apagar todo o restante.
